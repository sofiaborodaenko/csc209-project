1. Test that the game ends when a mine is selected
  
The input has a move following the selection of a mine. The output shows that the game ends in a loss, and the final move is not played. The board is a simple 3x3 grid with a mine in the middle. The first move selects the mine, and the second move selects an adjacent cell.
  
Files: test1_input.txt, test1_expected.txt
  
2. Test that the game ends with an inconclusive message when a mine is not selected and there remain safe cells that are not visible.
  
The board is a simple 3x3 grid with a mine in the middle. Cells (0, 0) and (0, 1) are selected leaving 6 safe cells not visible. The mine is not selected.
  
Files test2_input.txt, test2_expected.txt

3. Test game loss when a mine is revealed during sequential moves

The board is a 4×4 grid containing three mines. A sequence of moves is provided: the first two moves reveal safe cells, and the third move selects a cell containing a mine. The output shows the board state after each move, with numbers displayed for revealed safe cells and hidden cells represented by dots. After the mine is revealed, the game ends immediately with a loss message.

Files: test3_input.txt, test3_expected.txt

4. Test revealing cells on a single-row board with sequential moves

The board is a 1×5 grid containing two mines. A sequence of moves is provided that reveals cells from left to right. Each move progressively reveals either a zero-valued cell or a numbered cell, without selecting a mine. The output shows the board state after each move, separated by dashed lines. After all safe cells are revealed, the game ends with a win message.

Files: test4_input.txt, test4_expected.txt

5. Test progressive revealing and flood-fill behavior on a larger board

The board is a 5×5 grid with four mines placed at the corners. A long sequence of moves is provided that reveals cells across the board, including numbered cells and zero-valued cells that trigger flood-fill behavior. The expected output shows the board state after each move, demonstrating correct adjacency counts, correct stopping of flood fill at numbered cells, and correct handling of already revealed cells. After all non-mine cells are visible, the game ends with a win message.

Files: test5_input.txt, test5_expected.txt

6. Test flood-fill on a maximum-size board with no mines

The board is a 20×20 grid containing no mines. A single move is provided that reveals a cell with zero adjacent mines. Because the entire board is safe, the flood-fill reveals all cells on the board. The expected output shows the fully revealed board, followed by a win message indicating that all safe cells are visible.

Files: test6_input.txt, test6_expected.txt
